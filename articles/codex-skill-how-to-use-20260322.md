> この記事はAI(codex)で作成しています。

OpenAI公式のCodexドキュメントでは、`skill` は **Codexに新しい得意分野や作業手順を追加する仕組み** と説明されています。ひとことで言うと、毎回同じ説明をしなくても、Codexに「この仕事はこう進めてね」を覚えさせておける機能です。

たとえば、こんな使い方ができます。

- リリース前チェックのやり方を固定する
- ドキュメント更新の流れをまとめておく
- 特定のツールの使い方をCodexに覚えさせる
- チーム共通の作業手順を使い回す

## 何がうれしいの？

公式では、skillは「instructions」「resources」「optional scripts」をまとめたパッケージだとされています。つまり、ただのメモではなく、**手順書 + 参考資料 + 必要ならスクリプト** をひとまとめにできるのが強みです。

しかもCodexは最初から全部を読むわけではありません。まずは skill の名前や説明だけを見て、必要そうなときだけ `SKILL.md` を読み込みます。これにより、関係ない情報で会話が重くなりにくいのもポイントです。

## どうやって使うの？

公式では、skillの呼び出し方は2つあります。

1. 明示的に呼ぶ  
CLIやIDEでは `/skills` を使ったり、`$skill名` のように書いて呼び出せます。

2. Codexに自動で選ばせる  
作業内容が skill の説明文に合っていれば、Codexが自動で使うことがあります。

非エンジニアなら、最初は **明示的に呼ぶ** のがわかりやすいです。

```text
$openai-docs を使って、Codexの公式情報を調べて
```

```text
この作業では skill を使って進めて
```

## 自分で作ることもできる

OpenAI公式では、まず built-in の作成支援として `$skill-creator` を使うことが勧められています。何をする skill なのか、いつ使うのか、スクリプトが必要かを聞かれながら作れます。

手動で作る場合は、フォルダの中に `SKILL.md` を置くのが基本です。そこに少なくとも `name` と `description` を入れます。

## どこに置くの？

公式では、skill はリポジトリ内にも、自分用の保存場所にも置けます。

- リポジトリ用: `.agents/skills`
- 個人用: `$HOME/.agents/skills`

チームみんなで使いたいものはリポジトリ内、自分だけで使いたいものは個人用、と考えるとわかりやすいです。

## 追加インストールもできる

組み込み以外の skill を入れたいときは、公式では `$skill-installer` が案内されています。たとえば他のリポジトリ由来の skill を追加することもできます。

## まとめ

Codexのskillは、難しい拡張機能というより、**Codex用の再利用できる仕事の型** です。

覚えておきたいのはこの3つです。

- skill は Codexの得意分野や手順を追加する仕組み
- 明示的に呼ぶことも、自動で使わせることもできる
- よくやる作業を毎回説明しなくてよくなる

最初は、**「この作業は skill を使って進めて」** と頼んでみるだけでも十分です。慣れてきたら、自分専用やチーム専用の skill を作ると、Codexがかなり使いやすくなります。
