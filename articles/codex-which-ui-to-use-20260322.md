> この記事はAI(codex)で作成しています。

Codexを使おうとすると、まず迷うのがここです。

「CLIって何？」
「デスクトップアプリとWebはどう違うの？」
「結局、私はどれを使えばいいの？」

OpenAI公式のCodexドキュメントを見ると、Codexにはいくつかの使い方があります。でも、非エンジニア視点だと知りたいのは細かい仕様よりも、**自分に合う入口はどれか** だと思います。

この記事では、OpenAI公式の情報をもとに、Codexの主なUIをざっくり整理します。

## 先に結論

迷ったら、こんな感じで考えるとわかりやすいです。

- まず気軽に触りたい人: `Codex web`
- 見た目のある専用アプリがいい人: `Codex app`
- 普段からVS CodeなどのIDEを使う人: `Codex IDE extension`
- ターミナルに抵抗がない人: `Codex CLI`

ここから、それぞれをやさしく見ていきます。

## 1. Codex CLI

OpenAI公式では、Codex CLIは「ターミナルでローカル実行するCodex」と説明されています。自分のマシン上で、選んだディレクトリのコードを読んだり、編集したり、コマンドを実行したりできます。

要するに、**黒い画面でCodexと直接作業する版** です。

向いている人はこんな人です。

- ターミナルに抵抗がない
- ローカルのファイルをそのまま触ってほしい
- 細かい制御をしたい
- 自動化やスクリプト実行にも興味がある

逆に、非エンジニアだと最初は少しハードルが高めです。2026年3月22日時点の公式ドキュメントでは、CLIはmacOSとLinuxが主対象で、Windows対応はexperimentalと案内されています。

## 2. Codex app

Codex appは、OpenAI公式では「並列のCodexスレッドを扱うためのデスクトップ体験」と説明されています。ざっくりいうと、**Codexを使うための専用アプリ** です。

CLIより見た目がわかりやすく、複数の作業を整理しやすいのが特徴です。「作業を切り替えながら使いたい」「専用UIで管理したい」という人にはかなり相性がいいと思います。

向いている人はこんな人です。

- ターミナルはなるべく避けたい
- 専用アプリで落ち着いて使いたい
- 複数の作業を見やすく管理したい

これは公式の説明から見た私の整理ですが、**CLIほど玄人向けではなく、Webより“作業用アプリ”感が強い** 立ち位置です。

## 3. Codex web

Codex webは、ブラウザから使うCodexです。OpenAI公式では、Codex cloud taskをWebから起動し、リポジトリとつないでクラウド上で作業させる流れが案内されています。

つまり、**ブラウザからCodexに仕事を依頼する入口** と考えるとわかりやすいです。

向いている人はこんな人です。

- まずはインストール少なめで触りたい
- ローカル環境より、クラウド側に任せたい
- ブラウザ中心で使いたい

非エンジニアにとっては、最初の入口としてかなりわかりやすい選択肢です。まず試すだけなら、一番入りやすいと思います。これは公式のWebとCodex Cloudの説明をもとにした整理です。

## 4. Codex IDE extension

これは、VS CodeなどのIDEの中でCodexを使う形です。OpenAI公式では、IDE extensionを使って、エディタの中でCodexと並走したり、クラウドタスクへ委任したりできると説明されています。

要するに、**いつものエディタの中にCodexが入ってくる版** です。

向いている人はこんな人です。

- ふだんからVS Code系を使っている
- コードを書きながらその場でCodexに頼みたい
- エディタを離れたくない

すでに開発ツールに慣れている人にはかなり自然ですが、非エンジニアだと「IDEって何？」の段階だと少し遠回りかもしれません。2026年3月22日時点の公式ドキュメントでは、IDE extensionのWindows対応もexperimentalと案内されています。

## じゃあ、どれを選べばいい？

非エンジニア向けにかなり単純化すると、こうです。

- いちばん手軽に試す: Web
- 専用アプリで使いたい: App
- VS Codeの中で使いたい: IDE extension
- 細かく使いこみたい: CLI

Windowsユーザーは少し補足があります。2026年3月22日時点のOpenAI公式ドキュメントでは、CLIとIDE extensionのWindows対応は実験的という案内があります。一方で、appページにはWindows向けダウンロード案内があります。なので、Windowsで最初に触るなら、**Webかappから入るほうが素直** です。

## そのほかの使い方もある

ここまで紹介したのは、主に「人が画面から触るUI」です。OpenAI公式では、このほかにもCodexを使う入口があります。

- GitHub上から使う  
PR上でレビューを頼むような使い方です。GitHubの画面からCodexに作業を依頼するイメージです。

- GitHub Actionsで自動実行する  
CI/CDの流れの中で、Codexを自動で動かす使い方です。人が毎回手で操作しなくてもよくなります。

- SDKで自分のアプリに埋め込む  
自社ツールや自分のサービスの中から、プログラム経由でCodexを使う方法です。

ただし、非エンジニアが最初に触る入口としては、やはり **Web / app / IDE / CLI** の4つを先に押さえるので十分だと思います。

## まとめ

CodexのUIは、機能の優劣というより **使う人のスタイルに合わせて入口が分かれている** と考えるとわかりやすいです。

- Web: まず試したい人向け
- App: 専用アプリで整理して使いたい人向け
- IDE: 普段からエディタ中心の人向け
- CLI: ローカルで強く使いこみたい人向け

「どれが最強か」ではなく、**自分が一番ストレスなく触れる入口はどれか** で選ぶのがよさそうです。
