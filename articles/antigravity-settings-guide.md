## はじめに

Google が開発した AI コーディングエージェント **Antigravity**。VS Code 上で動作し、指示するだけでコードを書いてくれる優秀なやつです。

……が、**設定画面が全部英語**。

UIを日本語化しても設定画面だけは頑なに英語のままです。なぜだ。おまえのことは好きだが、そこだけは許せない。

この記事では、Antigravity の各設定項目を **日本語に翻訳** しつつ、それぞれの設定を選んだときの **メリット・デメリット** を解説します。

---

## 設定画面の開き方

![設定メニューの画像](https://raw.githubusercontent.com/harunamitrader/zenn/main/images/07_settings_menu.png)

設定画面を開くには、以下のいずれかの方法を使います。

1. **メニューバー**から「設定」→「**Open Antigravity User Settings**」をクリック
2. **ショートカットキー** `Ctrl + ,`（カンマ）を押す

:::message
VS Code の通常の設定（settings.json）とは別のウィンドウが開きます。
:::

![設定画面の全体像](https://raw.githubusercontent.com/harunamitrader/zenn/main/images/00_settings_overview.png)

## 🔒 SECURITY（セキュリティ）

### Strict Mode（厳格モード）

![Strict Modeの設定画面](https://raw.githubusercontent.com/harunamitrader/zenn/main/images/01_strict_mode.png)

**翻訳：**
有効にすると、エージェントが自律的に標的型エクスプロイト（悪用可能なコード）を実行することを防ぐ設定が強制され、**すべてのエージェントの操作に対して人間のレビュー（確認）が必要**になります。

ひとことで言うと、「AIが勝手に危ない操作をしないように、全部の操作を人間がチェックするモード」です。

| | **メリット** | **デメリット** |
|:---|:---|:---|
| **ON** | 最も安全。AIの全操作を逐一確認できる | 全操作に確認が入るため、作業速度が大幅に落ちる |
| **OFF** | 他の設定で個別に制御でき、作業がスムーズに進む | AIが意図しない操作を行うリスクがわずかに上がる |

:::message alert
Strict Mode を ON にすると、他の個別設定（Terminal や File Access など）よりも**優先**して「すべての操作に人間の確認」が必要になります。最も強力な安全装置です。
:::

## 📄 ARTIFACT（成果物）

### Review Policy（レビューポリシー）

![Review Policyの設定画面](https://raw.githubusercontent.com/harunamitrader/zenn/main/images/02_review_policy.png)

**翻訳：**
エージェントが「成果物（Artifact）」のレビューを求める際の動作を指定します。成果物とは、AIがより充実した会話体験を提供するために作成するドキュメント（計画書・報告書など）のことです。

ひとことで言うと、AIが作成した計画書やドキュメントを「毎回確認するか」「AIに任せるか」「全部スルーするか」の設定です。

| 選択肢 | メリット | デメリット |
|:---|:---|:---|
| **Always Proceeds**（常に続行） | AIが止まらないので作業が最速 | 安全でない内容や誤った内容をそのまま使ってしまうリスク |
| **Agent Decides**（AIが判断） | 重要な場面ではAIが確認を求めてくれるバランスの良さ | AIの判断が常に正しいとは限らない |
| **Asks for Review**（常にレビュー） | AIが作ったものを毎回確認でき、安心感が高い | 頻繁に確認が入り、作業のテンポが落ちる |

## 💻 TERMINAL（ターミナル）

![Terminalの設定画面](https://raw.githubusercontent.com/harunamitrader/zenn/main/images/03_terminal_settings.png)

ターミナル関連の設定は **4つ** あります。ここが一番項目が多く、そして**一番重要**です。AIがあなたのPCでコマンドを実行する権限に関わるからです。

### ① Terminal Command Auto Execution（ターミナルコマンドの自動実行）

**翻訳：**
AIがターミナルコマンドを実行する前に確認を求めるかどうかの設定です。

※ この設定の変更は、すでに処理中の応答には反映されず、新しいメッセージから適用されます。

| 選択肢 | メリット | デメリット |
|:---|:---|:---|
| **Always Proceed**（確認なしで実行） | 長時間の無人運転が可能。作業が止まらない | `rm` や `del` など危険なコマンドも即実行されるリスク |
| **Request Review**（毎回確認） | どんなコマンドが実行されるか事前に確認できて安心 | 毎回の承認操作が手間になる |

:::message
Always Proceed を使う場合は、下記の **Allow/Deny リスト** との併用がおすすめです。
:::

### ② Enable Shell Integration（シェル統合の有効化）

**翻訳：**
有効にすると、エージェントはIDEのシェル統合機能を使ってターミナルコマンドの実行を検知・報告します。無効にすると、エージェントは独自のシェルを使用します。反映にはアプリの再起動が必要です。

| | **メリット** | **デメリット** |
|:---|:---|:---|
| **ON** | IDE連携により正確なコマンド検知・報告ができる | 特になし |
| **OFF** | 特になし | コマンド実行の追跡精度が下がる可能性 |

基本的に **ON のまま** で問題ありません。

### ③ Allow List Terminal Commands（許可リスト）

**翻訳：**
許可リストに登録したコマンドは、AIが**確認なしで自動実行**します。

- **Unixシェル**：登録したトークン（単語）がコマンドの先頭から一致すればマッチ
- **PowerShell**：登録したトークンがコマンド内の連続する部分と一致すればマッチ

**使い方の例：** `npm run` を登録 → `npm run dev` や `npm run build` が自動実行されるようになる

### ④ Deny List Terminal Commands（拒否リスト）

**翻訳：**
拒否リストに登録したコマンドは、AIが実行する前に**必ず確認**を求めます。**許可リストよりも優先**されます。

**使い方の例：** `rm` を登録 → ファイル削除系コマンドは必ず確認が入るようになる

:::message
Allow リストと Deny リストを組み合わせれば「安全なコマンドだけ自動実行、危険な操作は必ず確認」という環境が作れます。
:::

## 📁 FILE ACCESS（ファイルアクセス）

![File Accessの設定画面](https://raw.githubusercontent.com/harunamitrader/zenn/main/images/04_file_access_settings.png)

AIがあなたのファイルにどこまでアクセスできるかを制御する設定です。

### ① Agent Gitignore Access（.gitignore ファイルへのアクセス）

**翻訳：**
`.gitignore` に記載されたファイル（通常はGitの追跡対象外にしているファイル）を、AIが自動的に閲覧・編集できるようにします。

| | **メリット** | **デメリット** |
|:---|:---|:---|
| **ON** | AIの作業範囲が広がり、設定ファイルなども含めた包括的なサポートが受けられる | `.env` 等に保存した機密情報をAIが読める状態になる |
| **OFF** | `.env` に書いたAPIキーやパスワードなどの機密情報をAIに見られない | AIが参照できる情報が制限され、作業効率がやや下がる場合がある |

### ② Agent Non-Workspace File Access（ワークスペース外ファイルへのアクセス）

**翻訳：**
現在開いているプロジェクトフォルダの**外にあるファイル**を、AIが自動的に閲覧・編集できるようにします。

| | **メリット** | **デメリット** |
|:---|:---|:---|
| **ON** | 複数プロジェクト間の参照など、広範な作業が可能になる | PC上のあらゆるファイル（認証情報含む）にアクセス可能になり、セキュリティリスクが高まる |
| **OFF** | プロジェクトフォルダ内にAIの操作範囲が限定され安全 | 別フォルダの参照ファイルなど、プロジェクト外の情報を活用できない |

:::message alert
この設定は**最も慎重に扱うべき項目**の一つです。ONにすると、AIがPC上のあらゆるファイルを読み書きできるようになります。
:::

### ③ Auto-Open Edited Files（編集ファイルの自動表示）

**翻訳：**
エージェントがファイルを作成・編集した場合、バックグラウンドで自動的にファイルを開きます。

| | **メリット** | **デメリット** |
|:---|:---|:---|
| **ON** | AIが何のファイルを触ったか一目でわかる。安全性・利便性ともに向上 | 大量のファイルを編集した場合、タブが増える |
| **OFF** | エディタのタブが増えすぎない | AIの変更に気づきにくくなる |

基本的に **ON 推奨** です。

## ⚡ AUTOMATION（自動化）

![Automationの設定画面](https://raw.githubusercontent.com/harunamitrader/zenn/main/images/05_automation_settings.png)

### ① Agent Auto-Fix Lints（リントエラーの自動修正）

**翻訳：**
有効にすると、エージェントは自分の編集によって発生したリントエラー（コードの文法・スタイルの問題）を認識し、ユーザーに指示されなくても自動的に修正することがあります。

| | **メリット** | **デメリット** |
|:---|:---|:---|
| **ON** | コードの品質が自動で維持される。手間が減る | 意図的に残したスタイルをAIが「勝手に」直す可能性がある |
| **OFF** | AIによる意図しない修正が入らない | エラーに気づかないまま放置される可能性 |

基本的に **ON 推奨** です。

### ② Auto-Continue（自動継続）

**翻訳：**
有効にすると、AIの応答が1回の制限に達した場合、自動的に続きを出力します。OFFの場合、「続けますか？」と確認が入ります。

ひとことで言うと、AIの回答が長くなって途中で切れたとき、「自動で続きを話す」か「人間に聞いてから続ける」かの設定です。

| | **メリット** | **デメリット** |
|:---|:---|:---|
| **ON** | 長いタスクがノンストップで進む。作業効率が良い | AIが間違った方向に進み続けてしまう場合がある |
| **OFF** | AIの作業を区切りごとに確認でき、方向修正しやすい | 毎回「続ける」ボタンを押す手間がかかる |

## 🧠 HISTORY（履歴・記憶）

![Historyの設定画面](https://raw.githubusercontent.com/harunamitrader/zenn/main/images/06_history_knowledge_settings.png)

### ① Conversation History（会話履歴）

**翻訳：**
有効にすると、エージェントは過去の会話にアクセスして、応答の参考にすることができます。

ひとことで言うと、「前に話したこと覚えておいてね」の設定です。

| | **メリット** | **デメリット** |
|:---|:---|:---|
| **ON** | 過去の会話の文脈を踏まえた回答が得られる。同じ説明を繰り返す必要がない | 過去の誤った情報を引き継いでしまう可能性がある |
| **OFF** | 過去の会話に引きずられず、毎回まっさらな状態で応答する | 毎回プロジェクトの背景説明が必要になり手間がかかる |

### ② Knowledge（ナレッジ）

**翻訳：**
有効にすると、エージェントはナレッジベース（知識データベース）にアクセスして応答の参考にし、バックグラウンドで自動的にナレッジアイテム（学習メモのようなもの）を生成します。無効にしても既存のナレッジアイテムは削除されませんが、アクセスできなくなります。

| | **メリット** | **デメリット** |
|:---|:---|:---|
| **ON** | 使い込むほどAIがプロジェクトに詳しくなる。回答の精度が上がる | 蓄積された情報が古くなっても参照され続ける場合がある |
| **OFF** | 不要な情報の蓄積を防げる | AIが毎回ゼロから理解し直す必要があり、効率が下がる |

## 🛠️ GENERAL（全般）

![Generalの設定画面](https://raw.githubusercontent.com/harunamitrader/zenn/main/images/08_general_settings.png)

全般的な挙動に関する設定です。使い勝手に直結する項目がいくつかあります。

### ① Open Agent on Reload（再起動時にエージェントを開く）

**翻訳：**
ウィンドウを再読み込み（リロード）した際に、自動的にエージェントのサイドパネルを開きます。

| | **メリット** | **デメリット** |
|:---|:---|:---|
| **ON** | 作業を再開する際、毎回パネルを開く手間が省ける | パネルが不要な時も勝手に開いてしまう |
| **OFF** | 画面を広く使いたい時に邪魔にならない | 毎回手動でパネルを開く必要がある |

### ② Auto-Expand Changes Overview（変更概要の自動展開）

**翻訳：**
エージェントが回答の生成を終えたとき、変更内容の概要（Changes Overview）ツールバーを自動的に広げて表示します。

| | **メリット** | **デメリット** |
|:---|:---|:---|
| **ON** | AIがどのファイルをどう変えたか、すぐに確認できる | 画面上の表示領域を占有してしまう |
| **OFF** | 生成完了後の画面がスッキリする | 変更内容を詳しく見るのにクリックが必要 |

### ③ Explain and Fix in Current Conversation（現在の会話で解説と修正を行う）

**翻訳：**
有効にすると、「Explain and Fix（解説して修正）」アクションが、新しいスレッドを開始するのではなく、現在の会話の中で継続されます。

| | **メリット** | **デメリット** |
|:---|:---|:---|
| **ON** | これまでの文脈を維持したまま修正を依頼できる | 回答が長くなり、ログが読みづらくなる場合がある |
| **OFF** | 新しい会話として切り出すため、トピックが整理される | 以前の経緯を一から説明し直す必要がある |

## おわりに

設定画面が英語のままなのは正直つらいですが、一度理解してしまえば大した量ではありません。

大事なのは「**自分がどれくらいAIを信頼するか**」と「**どこまで手間を減らしたいか**」のバランスで設定を決めること。迷ったらまずは安全寄りの設定から始めて、少しずつ自分好みに調整していくのがおすすめです。

この記事が、英語の壁に阻まれていた方の助けになれば幸いです。

**……でもやっぱり日本語化してほしいけどね、Google さん。**
