## はじめに

Google純正のAIエージェント、**Antigravity**。  
非常に強力ですが、初期設定のままだと「英語でしか返事をしてくれない」「勝手にコマンドを実行して不安」といった声もよく聞きます。

実はこれらの挙動は、設定画面から自分好みにカスタマイズ可能です。  
この記事では、Antigravityを「最高に使いやすい相棒」にするための主要な設定項目とそのメリット・デメリットを徹底解説します。

## ⚙️ 設定画面の全体像

Antigravityの設定画面は、大きく分けていくつかのカテゴリに分かれています。

![設定画面の全体像](/images/00_settings_overview.png)

右下の歯車アイコンから設定を開くと、左側のメニューから各項目を選択できます。今回は特に重要な項目をピックアップしてご紹介します。

---

## 🛡️ SECURITY（セキュリティ設定）

### Strict Mode（厳格モード）

AIが勝手に動くのをどこまで許容するか、という「ブレーキ」の強さを決める設定です。

![Strict Modeの設定画面](/images/01_strict_mode.png)

| 選択肢 | メリット | デメリット |
| :--- | :--- | :--- |
| **ON** | AIの全操作を逐一確認でき、最も安全。 | 全操作に確認が入るため、作業速度が大幅に落ちる。 |
| **OFF** | 他の設定で個別に制御でき、作業がスムーズに進む。 | AIが意図しない操作を行うリスクがわずかに上がる。 |

---

## 📄 ARTIFACT（アーティファクト設定）

### Review Policy（レビュー方針）

AIがコードやドキュメントを「作成・修正」した際、人間に確認を求めるかどうかの設定です。

![Review Policyの設定画面](/images/02_review_policy.png)

| 選択肢 | メリット | デメリット |
| :--- | :--- | :--- |
| **Always Proceeds** | AIが止まらないので作業が最速。 | 安全でない内容や誤った内容をそのまま使ってしまうリスク。 |
| **Agent Decides** | 重要な場面ではAIが確認を求めてくれるバランスの良さ。 | AIの判断が常に正しいとは限らない。 |
| **Asks for Review** | AIが作ったものを毎回確認でき、安心感が高い。 | 頻繁に確認が入り、作業のテンポが落ちる。 |

---

## 💻 TERMINAL（ターミナル設定）

### Terminal Command Auto Execution（ターミナルコマンド自動実行）

AIがコマンドを実行する際、人間に確認を求めるかどうかの設定です。

![Terminal設定画面](/images/03_terminal_settings.png)

| 選択肢 | メリット | デメリット |
| :--- | :--- | :--- |
| **Always Proceed** | コマンド（npm installなど）が自動で進むため、待ち時間なし。 | 誤った削除コマンドや破壊的な操作が実行される恐れ。 |
| **Asks for Review** | どんなコマンドが実行されるか事前に分かるので確実。 | 毎回の承認操作が手間に感じることがある。 |

---

## 📂 FILE ACCESS（ファイルアクセス設定）

### File Implementation Refactoring（リファクタリングの許可）

AIが直接ファイルを書き換える際の挙動を設定します。

![File Accessの設定画面](/images/04_file_access_settings.png)

| 選択肢 | メリット | デメリット |
| :--- | :--- | :--- |
| **Always Proceed** | 高速に次々とファイルを修正してくれる。 | 大規模な変更時に予期せぬバグが紛れ込むリスク。 |
| **Asks for Review** | 差分を慎重に確認してから適用できるため、安心。 | 修正ファイルが多い場合、ひたすらApproveを連打することになる。 |

---

## 🤖 AUTOMATION（自動化設定）

### Automation Mode（自動化モード）

タスク全体を自動でどこまで進めるかを決めます。

![Automationの設定画面](/images/05_automation_settings.png)

| 選択肢 | メリット | デメリット |
| :--- | :--- | :--- |
| **Always Proceed** | 寝ている間にタスクを完了させる、といった使い方も可能。 | 途中で意図しない方向に進んでも気づきにくい。 |
| **Asks for Review** | ステップごとに確認できるため、軌道修正が容易。 | 自分でポチポチ進める必要があり、自動化の恩恵が減る。 |

---

## 🧠 HISTORY & KNOWLEDGE（履歴と知識）

### Knowledge Search（知識検索の深度）

AIが過去の会話やコードベースをどれくらい深く調べて回答するかを設定します。

![History/Knowledgeの設定画面](/images/06_history_knowledge_settings.png)

| 選択肢 | メリット | デメリット |
| :--- | :--- | :--- |
| **High** | 過去の決定事項を考慮した、精度の高い回答が得られる。 | 検索に時間がかかり、回答までの待ち時間が増える。 |
| **Standard** | 回答速度と情報の正確さのバランスが良い。 | 非常にマニアックな過去の経緯までは拾いきれない場合がある。 |

---

## ⚙️ GENERAL（全般設定）

全般的な使い勝手を向上させる項目です。

![Generalの設定画面](/images/08_general_settings.png)

| 選択肢 | メリット | デメリット |
| :--- | :--- | :--- |
| **Explain and Fix in Current Conversation** | 文脈が維持されるため、エラー解説や修正がスムーズ。 | ログが長くなり、トークン（コスト）消費が増える。 |
| **Open Agent on Reload** | 再起動後に自動でパネルが開くため、即座に作業を再開できる。 | 画面を広く使いたい時に邪魔に感じることがある。 |
| **Follow Along by Default** | エージェントの動きを自動追従し、リアルタイムで把握しやすい。 | 意図しない時に画面が勝手にスクロールされることがある。 |
| **Auto-Expand Changes Overview** | どのファイルが変更されたかを一目で確認でき、差分チェックが捗る。 | 変更が大量にある場合、一覧の表示だけで画面が埋まってしまう。 |

---

## まとめ：どう設定するのが正解？

個人的には、**「最初は Asks for Review（毎回確認）** で慣れていき、信頼できるようになってきたら徐々に **Agent Decides** や **Always Proceed** に切り替えていく」のがおすすめです。

Antigravityは恐ろしく優秀ですが、あくまであなたの開発を支える「ツール」です。  
自分にぴったりの「ブレーキとアクセル」を見つけて、快適なAI開発ライフを送ってください！
